Offline Playback with DRM in the Brightcove Native Playback SDKs for iOS and Android allows publishers to reach their viewers in entirely new ways while ensuring their content remains secure. Our SDKs offer powerfully simple solutions for complex offline playback problems including download management, offline catalog management, analytics, playback, and of course DRM.

This topic contains the following sections:

  • Supported device versions
  • Security
  • Download management
  • Local catalog management
  • Analytics
  • Coming soon

 

Supported device versions

The Brightcove Native Player SDKs support offline DRM with the following device OS versions:

  • iOS 10.3+
  • The beta release is supported on Android 5+; The GA release will be supported on Android 4.4+

Security

The protection of downloaded content is top-of-mind with our Offline DRM solution. The Brightcove Native Player SDKs provide the following functionality to ensure your content is stored safely on viewers devices:

  • Your content is protected by Widevine Modular on Android and FairPlay on iOS.
  • Video content can only be watched on the device and app that has downloaded the content.
  • Video content can only be watched until a specific date and time expiration.
  • The Dynamic Delivery video URLs returned to the client will expire so that others can not use them.
  • When the app is deleted, all downloaded content is removed.
  • Your app can be configured to delete locally stored content at any time.

Download management

The Brightcove Native Player SDKs handle the complexities of downloading content to a viewer's device, greatly simplifying the process for native applications.

The following functionality can be added to your apps to support download management:

  • Display content
  • Show download status
  • Manage the download queue
  • Show local storage space
  • Pause/resume/cancel download
  • Playback content

Display content

Display a list of content available for streaming or download.

When you set a video's data parameter as downloadable using the CMS API, then the Native SDKs will return these videos from your calls to the Playback API.

Show download status

Display the current download along with its status.

Total estimated size, current download status and percent of downloading progress information will be returned by the SDKs.

Manage the download queue

Create and manage the download queue.

The Native SDKs download videos one at a time and provide updates so that you can manage the queue.

Show local storage space

Inform the user when there is not enough local storage to complete downloading the current item and any items in the queue.

The SDKs provide the estimated size and progress so that you can determine if there is sufficient space for downloads.

Pause/resume/cancel download

Allow the user to pause, resume or cancel content download.

Download functionality will resume where it left off IF the user pauses/resumes or if the download is interrupted. If download is cancelled, any progress made will be erased. For now, the Native SDK for Android only supports the cancel functionality.

Playback content

Playback content while it downloads. Downloaded content can be played back locally when the device is offline or streamed when the device is online.

The SDKs support this. The Application can choose to play the locally stored content or stream it instead.

Local catalog management

The Brightcove Native Player SDKs simplify the process of managing downloaded local content.

The following functionality can be added to your apps to support local catalog management:

  • Show downloaded content
  • Play local content
  • Delete local content

Show downloaded content

Show all of the downloaded episodes for each show/season.

The SDKs return metadata and playback details for all downloaded content. You can then filter the results for your specific implementation.

Play local content

Allow users to playback local content.

The SDKs accept local content as a playable source.

Delete local content

Allow users to delete content from local storage.

The SDKs expose a method to delete a video along with its metadata and thumbnail images.

Analytics

There are two new types of analytics being introduced for measuring data related to the Offline flow: Download analytics & Offline playback metrics.

Download analytics

Download Analytics will automatically track:

  • Every download request - This includes rendition bitrate and rendition dimensions
  • Every download complete
  • Every download failure
  • Every download cancellation

For details about viewing these download metrics from the Analytics API, see the video Dimension document.

Offline playback metrics

Offline Playback Metrics will track various playback data, however is not yet available.

Coming soon

  • For now, publishers will not be able to specify download quality. The Native SDKs will use the highest rendition associated with the requested video. In the future, you will be able to specify the maximum average bitrate for the downloaded video.
  • In the future, download and display of closed captions will be supported.
  • In the future, playback of multiple audio tracks will be supported.
  • The download queue should be handled at the app level.
