Overview

Brightcove Live provides a robust service for creating live streaming events or 24/7 live streams. This guide outlines best practices for optimizing your live streams

Input bandwidth

Providing a high-quality, stable input stream is the only way to ensure the best user experience for viewers. A good input stream provides the best video quality at the highest consistently available bandwidth from a location.

Supported formats

Input

For best results, use H.264 and AAC codecs in your RTMP stream. We also support VP6 and MP3, but quality may not be as high when using these codecs. Sending Brightcove Live the highest quality stream that your upstream connection supports will improve the quality of the streams generated.

Output

Live output videos can be published to an RTMP server or uploaded as HLS segments. Regular, non-live files can also be generated when the input stream has ended.

FAQ

How soon do you have to start streaming after creating a live job?
By default, the job will finish 30 minutes after it is created if you do not connect a live stream. You can change this value by setting the reconnect_time value when you create the job. To prevent the job from finishing unless you finish it manually, set the reconnect_time to 0.
