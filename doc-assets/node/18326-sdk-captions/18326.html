In this topic, you will learn how to use captions with apps built with the Brightcove Native SDKs.

Understanding captions

Closed captions can be added to your videos to provide additional information about your video.

First, it is important to understand which types of captions work best for each video format. Begin by reviewing the Video Formats and Captioning document. Brightcove recommends using HLS videos with WebVTT captions.

Adding captions to videos

Learn how to create a captions file and add it to your videos:

  • Add captions to videos using the Media Module
  • Add captions to videos programmatically

HLS Video

With HLS video, there are two options for captions playback:

  • In-manifest WebVTT
  • External WebVTT

In-manifest WebVTT

When using In-manifest WebVTT captions with HLS videos, your captions will just work. There is nothing extra that you need to do to implement these captions. If you are using Video Cloud and want to use this type of captions, then your videos need to be ingested with in-manifest WebVTT captions. For details, see the In-Manifest WebVTT section of the HLS Plugin document.

External WebVTT

External WebVTT captions are text files separate from the video.

Android

The WebVTTSampleApp shows you how to add WebVTT captions files for video playback.

The DFXPClosedCaptioningSampleApp demonstrates playback and control of DFXP/TTML captions with videos retrieved from your Video Cloud library.

iOS/tvOS

When using external WebVTT captions, you need to include the Sidecar Subtitles plugin for the Player SDK for iOS and tvOS.

If you are retrieving videos from your Video Cloud library using the BCOVPlaybackService, the videos and text tracks will be formatted for the Sidecar Subtitles plugin.

If you are providing your own videos or you are a Brightcove Player customer without Video Cloud, then you need to structure the text tracks as shown in the Manually populating subtitle data section of the Sidecar Subtitles documentation.

Synchronizing WebVTT captions

If you are using WebVTT captions with HLS video, make sure the video and audio is synchronized with the captions. Learn how to do this in the Synchronizing WebVTT Captions document.

Styling captions

WebVTT captions can be styled to match your company preferences. For details, see the Styling captions section of the Add Captions to Videos document.

Combining plugins

Most plugins for the Native Player SDKs can be combined, so for example, you could have captions with a DRM-protected video that also plays Google IMA ads.

Android

With the Native SDK for Android, most plugins can be combined. This sample app shows you how to use the Brightcove ExoPlayer with Google IMA ads and Widevine Modular content.

iOS/tvOS

With the Native SDK for iOS, the BCOVSessionProviders from each plugin are created and chained together. This chain is then used to create the BCOVPlaybackController. For details, see the Combining Plugins section of the SDK Reference.

Example

If a video has captions associated with it, you will see a captions icon in the control bar. Select this icon.

Then, choose your preferred language.

During video playback, you should see captions displayed.
