Working with Captions

Introduction

"Captions" here is used to mean bits of text provided by an external file that are overlaid on a video, as seen in the screenshot below:

Such text may be used for accessibilty or to provide translations of the audio track, or some other purpose, but the underlyting technology used is the same.

New and old players

Although captions look similar in the Brightcove Player and the legacy Smart Player, the underlying technologies are significantly different:

Captions in the Brightcove Player and Smart Player
Player Captions format File extension Cross-domain control
Brightcove Player WebVTT (text tracks). (Note that WebVTT is a general syntax for providing information about time-based events during video playback, and may be used for other purposes such as specifying video chapters as well.) .vtt CORS
Smart Player (legacy) DFXP .xml crossdomain.xml

The remainder of this document deals with WebVTT captions (text tracks), since that is the technology used for the current Brightcove Player. Additional information about captions for the legacy Smart Player can be found here.

Add captions to video

You can associate a WebVTT file with a Video Cloud video to provide captions in two ways:

  1. Use a remote asset, meaning that the WebVTT is hosted on your server, and you provide add the URL for the file to the video metadata.
    Note: If you are hosting captions files on your own server, make sure the file is located at a URL that can be accessed by the Brightcove player. The server hosting the captions file should also have Cross-Origin Resource Sharing (CORS) enabled. You can read more about CORS here.
  2. Ingest the WebVTT file to Video Cloud.

The following sections will point to the relevant documentation for these two options.

Add captions in Studio

Both remote and ingested caption files can be added through the Media module in Studio. See Configuring the text track properties.

Add captions via the APIs

How you add captions via the APIs varies depending on whether the captions file will be ingested or used as a remote asset.

There is a /caption endpoint for the CMS API, but this is used only for DFXP captions for the Smart Player.

Remote asset captions

If you prefer to host the captions yourself (or have a captions provider who hosts them), then you can associate the captions with a video using the CMS API Update Video and including the text_tracks array in your request body.

Add captions to Video Cloud

In you want to have Brightcove host your captions, you can ingest your WebVTT file via the Dynamic Ingest API.

Removing captions

In Studio

WebVTT captions can be removed from videos in Studio - see Configuring the text track properties for details.

Via the CMS API

Remove WebVTT captions from a video by making an Update Video request and submitting a request body with an empty text_tracks array (or an array from which the object associated with that set of captions has been removed).

Styling captions

Styling can be appliedt to WebVTT captions using CSS. See Styling Captions for details.
