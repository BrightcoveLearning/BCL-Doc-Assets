New Video Cloud options

Note: this topic applies only to Ooyala customers migrated to Video Cloud.

Video Cloud has three standard options for ingesting videos:

  • The Upload module in Studio
  • Pull-based ingestion using the Dynamic Ingest API
  • Source file upload using the Dynamic Ingest API

As a Video Cloud customer, you can use any of these options, but Brightcove has also built adaptors for Ooyala ingest systems to direct the videos to the Video Cloud ingest system, so you can continue to use the Ooyala Ingestion Service (OIS) to send your videos to Video Cloud using:

Or use Ooyala ingestion methods

As a Video Cloud customer, you can use any of the options listed above, but Brightcove has also built adaptors for Ooyala ingest systems to direct the videos to the Video Cloud ingest system, so you can continue to use the Ooyala Ingestion Service (OIS) to send your videos to Video Cloud using:

  • FTP / FTPS
  • Aspera
  • Remotely host MRSS feeds

All existing manifest options will continue to work. The only change you will need to make is authentication for secure sources such as SFTP and Aspera. You will receive client credentials (a client_id and client_secret that you will use as the username/password for authentication.

You can also create additional client credentials if you need them, using either the Studio admin page or the OAuth API. For more information, see Authentication for Video Cloud Platform APIs.
