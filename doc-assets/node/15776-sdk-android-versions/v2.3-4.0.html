The Brightcove Player SDK for Android will be removing support for versions 2.3 and 4.0 as of January 1st, 2016. Google continuously releases content updates to Android which affect the technology that the Brightcove SDK works with. In order to focus on cutting edge features we must evolve alongside Google by providing support only for relevant versions of Android.

Data

Support is defined in terms of the major.minor.revision of an OS (for example: Android 2.3.3). Only the latest revision of a version is supported. Customers can still use the Brightcove SDK for Android on unsupported versions, however we provide no guarantee of functionality.

Google Play Store

Google’s Play Store provides usage data that shows which Android versions have logged into the Play Store for a highlighted week. As of November 2nd 2015, data shows that 92.7% of devices connected to the Play Store are running Android 4.1 or higher. Setting the drop date for January gives us several more months for that remaining 7.3% to decrease. Typically, the holidays see a dramatic increase in more modern versions as people upgrade their devices. Current Play Store data can be found here.

Brightcove SDK for Android Streams

In addition to examining the Play Store data we can also review traffic that the Brightcove SDK is seeing across the different Android versions. This is much more relevant data as it gives us an idea of where video is actually being watched.

The above chart shows a movement towards recent Android versions as 98% of Brightcove video views are coming from Android 4.1+; Android 2.3 doesn’t even appear on the scale. This data complements the Play Store analysis and shows that even though there are still a few 2.3 and 4.0 devices out there, they aren’t being used to watch content through our SDKs.

The cost of supporting older Android versions

Committing to older versions comes with heavy maintenance and upkeep costs which outweigh the benefits of supporting a minute fraction of the market. The following are a few considerations that the team faces supporting older versions:

  • Bug fixes released in newer versions of Android still need to be accounted for in older versions of our SDK.
  • Features released in newer versions of Android enable us to create functionality we could not otherwise easily build. This forces us to forgo taking advantage of new features in Android, or to spend the time building up feature parity within the SDK for older versions.
  • Partners often focus on supporting more recent versions which causes issues for customers expecting integrations to work on older devices.

End of life schedule

In preparation for this transition we will be marking Android 2.3.x and 4.0 as deprecated on our documentation sites and in the project changelog. As of January 1st 2016 we will remove these versions from our supported devices list.
