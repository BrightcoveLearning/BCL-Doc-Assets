After completing this Quick Start, you should be able to:

  • Use the Video Cloud Live module to create a live event
  • Configure your encoder for a live event
  • Trim your event and save it as a VOD asset in your Video Cloud account
  • Monitor the health of your live streams

This Quick Start will help you broadcast a live event to different devices using the free Adobe Flash Media Live Encoder. Other encoders can also be used with the Live module.

Audience

Any Video Cloud publisher who needs to broadcast a live streaming event using the Video Cloud Live module.

Prerequisites

  • A Video Cloud Pro or Enterprise account with access to the Live module; you can sign up for a Live module trial here
  • A camera to broadcast the event
  • You must have your own encoding software; this Quick Start uses Adobe Flash Live Media Encoder which can be downloaded for free (see a list of supported encoders)

Limitations

  • Your encoder must output an RTMP stream
  • Live broadcasts will experience a 20-30 second delay
  • Maximum of 26 continuous event hours
  • Streaming must begin within 30 minutes of the event start
  • HD is defined as video with a height of 720 and up; SD is video with a height below 720
  • Akamai HD must be used as the Content Delivery Network (CDN)
  • For users using Internet Explorer, version 11 or later is required (Internet Explorer 10 will work with Player version 5.15.0)

Using the Video Cloud Live module to broadcast a live event

To stream your live event using the Live module, you will complete the following tasks:

  • Enter your event information using the Video Cloud Live module
  • Configure your encoder
  • View the live stream
  • Trim and save the stream to your Video Cloud account

Enter your event information using the Video Cloud Live module

  1. In a browser window, navigate to Video Cloud Studio.
  2. Enter your Video Cloud login credentials and click Sign In.
  3. The current Video Cloud account name will appear in the upper right corner of the page. If you have multiple accounts, click the account selector drop down and select the account you would like to save the live asset in. If you select an account that has not been enabled for Live, the Live module will open in trial mode.
  4. Click Live in the navigation header.
  5. Click the Add An Event button to create a new event.
  6. On the Create Your Event dialog, add details for your live event and click Save and Continue. Note that the broadcast name, short description, tags and reference ID will be associated with the live event asset in Video Cloud.
    • Event Name - top level name for the event; this name is internal to the Live module
    • Broadcast Name - name for the broadcast

      Note: It is possible for an event to have multiple broadcasts but using multiple broadcasts is only recommended when using a Smart Player. If you are publishing an event using a Brightcove Player, Brightcove recommends creating new events. In this case, using the reference id in place of the video ID in the publishing code will allow you to use the same publishing code for different events. For information on working with Reference IDs, see Using Reference IDs.

      Note: While it is possible to click New Broadcast after an event has completed to start a new broadcast, Brightcove has seen cases where when using a Brightcove Player, old stream content displays instead of the current stream. Brightcove recommends creating new events vs. creating a new broadcast.

      Note: When using the reference ID in the player publishing code, Brightcove recommends that there is at least one hour between events due to the Playback API cache.

    • Short Description - description of the event
    • Tags - tags to associate with the event
    • Reference ID - reference ID associated with the event
    • Event Date - planned date of the event; for informational purposes only and has no effect on the event
  7. On the Select a Player dialog, select the player to use for the event. The drop down list will display the single video players in your Video Cloud account. Both Brightcove Players and Smart Players are supported. Brightcove Players will have (New Brightcove Player) following the player name. This quick start will use a Brightcove Player. Click Save and Continue.

    Note: The copy to clipboard buttons require that Flash is installed and enabled. These buttons do not work when using the Microsoft Edge browser.

  8. The Your Output Options dialog allows you to select a predefined transcoding profile for the event. Each profile has a set of transcode settings associated with it. Select a transcoding profile and then click View Details link to see the rendition details.

    Note: The transcoding profiles that appear are based upon your Live module subscription plan.

    Note: For information on creating custom ingest profiles including how to adjust the amount of DVR time, see Creating Transcoding Profiles for the Live Module.

  9. The Brightcove Player does not support RTMP renditions. In an earlier step a Brightcove Player was selected so we will create a custom profile with only HLS renditions. Click Edit Profile.

    Note: If using a Smart Player, RTMP renditions are supported.

  10. Change the RTMP renditions to HLS. If any renditions are similar, you may want to delete them or edit the Video Bitrate. For this quick start, the following renditions will be used.
  11. Change the profile Name to Live - HD Brightcove Player. Click Save As New.
  12. Click Start Streaming. The Live module will return a Stream URL and Stream Name. These values will be used in the encoder settings. If you Show the Live Event Player information, you will see a URL which you can use to access your live stream. In the Live Event Player pane, you can choose to copy the URL or the embed code for the live event player.

    Note: After creating a new live event, an asset for the live event will be created inside of the Media module. This asset SHOULD NOT be deleted until the event has finished streaming. Deleting the live asset before the event has started streaming will cause errors inside of the Live module.

    Note: The copy to clipboard buttons require that Flash is installed and enabled. These buttons do not work when using the Microsoft Edge browser.

Configure your encoder

This quick start uses Adobe Flash Media Live Encoder. For a list of other supported encoders, see the Zencoder documentation.

  1. Launch Adobe Flash Media Live Encoder.
  2. Configure the Video encoding options. Use the following guidelines when setting the rendition properties. Keep all other values at their default.

    Note:  For the best quality, we recommend configuring your encoder to output a 1280 x 720, 2500 Kbps stream.

    • Device: Select the camera you are using for the live stream. If no cameras appear, make sure your camera is attached to your computer.
    • Format: H.264
    • Frame Rate: 30 fps
    • Input Size: 1280 x 720
    • Bit Rate: 2500 Kbps
    • Output Size: 1280 x 720
    • Audio:
      • Format: AAC
      • Channels: Stereo
      • Sample Rate: 48000 Hz
      • Bit Rate: 128 Kbps
  3. Configure the encoding Output options. Use the following guidelines when setting the output properties. Keep all other values at their default.
    • Stream to Flash Media Server: checked
    • FMS URL: Use the Stream URL value returned by the Live module
    • Backup URL: leave blank
    • Stream: Use the Stream Name value returned by the Live module
    • Save to File: "Check" if you want to save the contents of the live stream to a local file

      Note: While the Live module will keep an archive of the stream and allow you to trim and save it to a video on demand title for 7 days, it is recommended that you save a local copy as a redundant backup.

  4. Click the Start button to start the live stream. Check the bottom left of the media encoder dialog to verify that you have connected.

Note: It will take about a minute to process your streams.

View the live stream

  1. To view the live stream, you can:
    • Scan the QR code and preview the stream on a mobile device
    • Copy the link next to the QR code and open it in a browser
    • Expand the Live Event player section , copy the URL to the player and open it in a browser

      Note: If RTMP outputs are available a preview player will appear in the page.

      Note: The Flash plugin must be enabled for the preview player to display the live stream. If Flash is not enabled, a Processing Stream message will appear and the live preview will not display. When using Google Chrome, the Processing Stream message may appear instead of the live stream preview.

  2. Confirm that you can see the live stream.
  3. When you are done with the live stream, click the Stop Streaming button in the Live module. Also, return to the Flash Media Live Encoder dialog and click Stop to stop the encoder from streaming.

    Note: It is important to make sure you close/stop your encoder when the live stream is complete. Leaving a connection open between any encoder and Brightcove, even when no data is being sent, may result in unexpected connection charges.

Trim and save the stream to your Video Cloud account

  1. When you have stopped streaming your event, the Live module offers the ability to trim and save your live event to your Video Cloud account. Click the Create Clip button in the Broadcasts section of the page. The Download Archive link allows you to download an archive of the event.

    Note: The option to trim and save the live event to a separate video is only available for 7 days. After that, the Create Clip button will not be available and there will be no way to archive the live event.

    Note: Archived live streams will be removed from the CDN after 7 days. If you want to keep your live content for longer than a week, we recommend that you save the event as a video-on-demand asset in your Video Cloud account.

  2. On the trimming dialog, drag the handles that appear on the timeline to trim the beginning and end of your event. You can also enter specific In and Out times using the edit boxes. When you are done, click Next.
  3. Enter a Video Name, Short Description, Tags and Reference ID and then click Save. These values will be associated with the trimmed asset in Video Cloud Studio. Click Save.

    Note: Clips are not processed if the video title contains double-byte characters.

  4. The video will be transcoded and saved into your Video Cloud account. When the transcoding is complete, expand the broadcast details and click on the Preview Video button to preview your live event in a player.

    Note: You can click the Create Clip button multiple times to create multiple clips of your broadcast. This allows you to trim and save as many times as you want for each broadcast, saving the clips as separate VOD assets in your media library.

Troubleshooting live stream issues

If you are having issues with your encoder, for example, if you receive a "Failure to connect to primary server" error, make sure that port 1935 is open on your network. Zencoder uses RTMP to accept the input stream and it uses this port.

The Video Cloud Live module can be used to view stream information and can be helpful when troubleshooting live stream errors. Brightcove Support is also available to help you troubleshoot live stream issues.

View a live stream preview using the Live module

You can view a preview of the live stream using the Live module.

  1. All on air events will display a thumbnail at the top of the Live module page.
  2. Click a thumbnail to view the live event status and a preview of the event. If RTMP renditions are available, a preview of the stream will appear.

    Note: The Flash plugin must be enabled for the preview player to display the live stream. If Flash is not enabled, a Processing Stream message will appear and the live preview will not display.

For further help

If you need further help getting your live event to work, you can contact us. To make sure you get the fastest response possible, below is a list of what support will need to solve the problem.

  • The specific symptoms the stream is having. For example, does it not play at all or does it stutter or freeze?
  • Whether this stream worked correctly in the past
  • The entry point URL you are using in your encoder
  • The stream name and Zencoder Job ID
  • The encoding software and hardware are you using
  • The URL to the player to which you have published the live event
  • The video ID of your live asset in Video Cloud Studio
  • The results of a trace-route from your encoder to the publishing point host
