The Social API is a REST API that allows you to manage the sharing of your Video Cloud videos to social media destinations such as Twitter and Facebook from your own CMS or app. Currently, the API provides operations to get the status of social sharing for videos.

Base URL

The base URL for the Social API is:

https://social.api.brightcove.com/v1

Authentication

Like most of Brightcove's REST APIs, the Social API uses the Brightcove OAuth service to authenticate API requests. This involves:

  • A one-time operation of creating client credentials consisting of a client id and a client secret
  • Using the client id and secret to obtain a temporary access token

The access token is passed in an Authorization header with the request:

Authorization: Bearer {access_token}

The easiest way to get credentials in most cases is through the Studio Admin API Authentication section (requires admin permissions on your account). See Managing API Authentication Credentials for details. In most cases, you probably just want to get permissions for all Ingest Profiles API operation:

If the permissions you need are not available in Studio, or if you prefer to get them directly from the OAuth API, use your choice of the Get Client Credentials documents listed below. Whichever option you choose, you will need to ask for the correct operation permissions. The following can be used with cURL or Postman to get all permissions for the Social API:

    "operations": [
      "video-cloud/distribution/destination/read"
    ]

Note on responses

Note that response for status requests may return HTTP 202 NO CONTENT if the video requested has not been distributed at all, or to the requested destination.

Available endpoints

The sections the follow provide brief descriptions of the endpoints currently available for the API. Full details of the endpoints, along with descriptions of parameters, responses, and errors, can be found in the Social API Reference

Get status of a video

Gets the current status of the requested video on every destination to which Social has ever attempted to distribute it, ordered by date, most recent first.

Method: GET

Endpoint: /accounts/:account_id/social-status/video/:video_id

Get Status of Videos for Destination

Gets the current status of every video which Social ever attempted to distribute to the given destination.

Method: GET

Endpoint: /accounts/:account_id/social-status/destination/:destination_id
