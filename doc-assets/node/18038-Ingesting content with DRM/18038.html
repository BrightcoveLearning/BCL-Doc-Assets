Overview

This topic deals with security that can be configured in your ingest profiles:

  • DRM
  • HLS encryption (HLSe)

For additional content security mechanisms that can be set in either the player configuration or video properties, see:

  • Configuring Player Availability
  • Configuring Video Availability

DRM is currently not supported on any mobile OS browsers, but Widevine is supported for Android apps and FairPlay is supported for iOS apps. For details about creating mobile apps, see the Brightcove Native Player SDKs index page.

A DRM player should not be used for mobile devices (iOS or Android) since it would prompt to install SilverLight. Workaround: Two players can be created, one with DRM support and one without, and switch between them based on the platform/video.

DRM

DRM (Digital Rights Management) protects your video content by encrypting the video data and unlocking it based on license policies. When the video content is loaded into a Brightcove player, the player calls back to a licensing server and obtains permission to play the video. This transaction is transparent to viewers.

Note: to use DRM or HLS encryption, your account must be enabled for these features. Contact your account manager for more information.

Video Cloud currently supports the several DRM types for different video formats:

DRM for Dynamic Delivery

If your account is enabled for Dynamic Delivery, then all you need to do to DRM-protect your videos is contact Brightcove Support to configure your account for the kinds of DRM packaging you want.

If your account is enabled for DRM and you want to deliver some titles without DRM, set the drm_disabled flag via CMS API to true using an Update Video request.

Sample request

Request

The request is PATCH request to:

    https://cms.api.brightcove.com/v1/accounts/account_id/videos/video_id

Request body

    {
        "drm_disabled": true
    }

You can also turn off DRM for particular videos by going to the video properties in Studio and clicking the DRM toggle:

After you click the toggle, it will look like this:

You do not need to create custom ingest profiles, and the remaining sections of this topic do not apply to you.

DRM for legacy ingest

If your account is not enabled for Dynamic Delivery, you must create custom ingest profiles to specify how you want your renditions packaged for DRM. This is explained in the sections that follow.

Reference IDs

Renditions are assigned to DRM packages by reference_id, so you need to be sure that you assign a reference_id to each rendition defined in profiles that will have DRM packages.

Example

..."renditions": [
    {
        "media_type": "video",
        "reference_id": "av0",
        "format": "mp4",
        "audio_codec": "aac",
        "width": 360,
        "height": 270,
        "h264_profile": "baseline"
    }...
]
Every reference_id must be unique across all profiles defined for your account.

Defining packages

For each DRM package you want to define, add an object to packages array specifying the package type and an array of rendition reference_ids to include in the package:

"packages" : [
    {
        "package_type" : "widevine",
        "renditions" : [ "wv0", "wv1","wv2", "wv3","wv4", "wv5","wv6", "wv7" ]
    },
    {
        "package_type" : "hds-faxs",
        "renditions" : [ "hds0" ]
    },
    {
        "package_type" : "hds-faxs",
        "renditions" : [ "hds1" ]
    },
    {
        "package_type" : "hds-faxs",
        "renditions" : [ "hds2" ]
    }
]
The example above is for the deprecated Widevine Classic - see the example below for packaging MPEG-DASH content with Widevine Modular.

For MPEG-DASH renditions, the syntax is slightly different, including an array of DRM types and dash as the package type:

"packages" : [
    {
        "drm" : ["widevine", "playready"],
        "package_type" : "dash",
        "renditions" : "av01"
    }
]
In the JSON for the profile, the renditions array must be defined before the packages array, so that the reference_ids referred to in the packages exist.

Widevine

Google's Widevine DRM packaging comes in two flavors: Modular, for MPEG-DASH video, and Classic, for WVC video. Modular is the newer type and recommended.

Widevine Modular

Widevine Modular is for MPEG-DASH video.

Notes

  • For MPEG-DASH renditions using Widevine Modular DRM, each rendition must include the property "streaming_delivery_profile": "live". Otherwise, Widevine protection will not be applied to the rendition.
  • The Brightcove player currently supports DASH playback only with the DRM plugin (even if you are not using DRM packaging):
    • //players.brightcove.net/videojs-drm/3/videojs-drm.min.js
    • //players.brightcove.net/videojs-drm/3/videojs-drm.css
  • The force_aac_profile property has to be present in each DASH rendition.
    • force_aac_profile must one of these three values:
      • aac-lc
      • he-aac
      • he-aac-v2
    • The value for force_aac_profile must be the same for all renditions in a profile
Widevine does not support video with no audio track.

Playready

Playready can be used to package MPEG-DASH renditions.

Notes

  • The Brightcove player currently supports DASH playback only with special plugins.
  • The force_aac_profile property has to be present in each DASH rendition.
    • force_aac_profile must one of these three values:
      • aac-lc
      • he-aac
      • he-aac-v2
    • The value for force_aac_profile must be the same for all renditions in a profile
