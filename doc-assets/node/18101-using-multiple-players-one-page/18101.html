Multiple players

If you are using multiple players on the same page, there is nothing special you need to do. For instance, it is perfectly acceptable to have the following players on the same page.

Note: If you are a Brightcove Player customer, you will not use the data-video-id attribute to assign a video to your player. You would either assign the video in the Players module or using curl.
<video data-video-id="4093372393001"
  data-account="1507807800001"
  data-player="ry3j6rsze"
  data-embed="default"
  data-application-id
  class="video-js"
  controls></video>
<script src="//players.brightcove.net/1507807800001/ry3j6rsze_default/index.min.js"></script>

<video data-video-id="5076962725001"
  data-account="1507807800001"
  data-player="BkTGbgSq"
  data-embed="default"
  data-application-id
  class="video-js"
  controls></video>
<script src="//players.brightcove.net/1507807800001/BkTGbgSq_default/index.min.js"></script>

 

You can use the Standard (iframe) code for multiple players on the same page also.

Special considerations

Be aware of the following when placing multiple players on the same HTML page:

  • You can use multiple instances of the same player on same page, using a single index.min.js file.
  • It can be very expensive bandwidth-wise to have multiple players on a page and have the preload set to auto. It is generally not recommended.
  • In Google Chrome, you cannot have the same video source in multiple players on the same page.
  • If you're using IE11 and have multiple instances of the same player on a page, you need to either be sure each player instance has a unique value for the id attribute or use no id attributes.
