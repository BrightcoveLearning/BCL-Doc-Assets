This topic applies when using the new Players module UI. For the legacy UI, see Configuring Player Settings.

The player playback properties can be used to configure how video playback is handled in the player.

To configure the player playback properties, follow these steps:

  1. Click Players in the navigation header to open the Players module.
  2. Click the link for the player you want to configure.
  3. Click Playback in the left navigation menu.

Note: Any changes made to a player may take up to 5 minutes to appear in published players.

The following playback properties can be configured:

  • Auto-Start Video on Player Load
  • Preload Video Before Playback

Auto-Start Video on Player Load

The Auto-Start Video on Player Load setting controls whether the video will start playing when the player is loaded.

Browser-specific limitations

  • Safari - Starting with Safari 11.0 (the standard version in MacOS 10.13 High Sierra), autoplay will not work. Using the API play() method will also not work until the user has initiated playback.
  • Chrome - Excerpt from blog.chromium.org/2017/09/unified-autoplay.html

    Starting in Chrome 64, autoplay will be allowed when either the media wonâ€™t play sound, or the user has indicated an interest in the media. This will allow autoplay to occur when users want media to play, and respect users' wishes when they don't. These changes will also unify desktop and mobile web behavior, making web media development more predictable across platforms and browsers.

    Not all users have the same preferences for autoplay media, so Chrome 63 will add a new user option to completely disable audio for individual sites. This site muting option will persist between browsing sessions, allowing users to customize when and where audio will play.

Preload Video Before Playback

The Preload Video Before Playback setting controls whether the player will preload video content before the user initiates playback.

Note: For more details, see the preload details section of the Available Attributes document.

The following options are available:

  • Preload Video (fastest) - Indicates that the whole video file could be downloaded, even if the user does not eventually watch the video. On the positive side, if the viewer plays the video, it can start playing immediately. On the negative side, this option will increase bandwidth consumption as the preload/load happens even if the viewer does not play the video.
  • Preload Metadata - Only the video metadata is preloaded. This is the default.
  • Do not preload - None of the video is preloaded

The value for the preload setting in Studio will override the value of the preload attribute value in the Advanced (In-Page) Embed Code.
