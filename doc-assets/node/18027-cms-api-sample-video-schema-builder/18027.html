Use and limitations

You can generate the code here and use it as your publishing code if you are using a single Brightcove player for the account and a standard (iframe) embed. If you are using another player or the advanced (in-page) embed, you can just use the schema part of the generated code and insert it above your player embed code in the page.

This is a client-side app, and and therefore makes requests to the CMS API from the client-side. However, you cannot simply use this script in pages with your players to generate effective schemas, because the schema code must be injected into the page on the server side, before the page is delivered to the browser, to be recognized by search engines. If you are serving your pages via a web server application, use the logic here to get the information using the CMS API and embed it in the page before it is served. The schemas generated by this app have to hard-coded into the player page.

Also note that this app produces code for a single video player only, and if you are embedding the schema code manually, you should update it periodically, as URLs can change
