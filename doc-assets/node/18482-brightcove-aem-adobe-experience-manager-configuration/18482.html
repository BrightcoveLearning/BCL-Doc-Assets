Introduction

The Adobe-AEM-Brightcove Connector allows you to manage Brightcove Video Cloud videos and players within AEM, and easily embed videos in AEM pages.

Note that this guide uses AEM 6.3. The connector also works with AEM 6.2, and the steps are similar.

Configuration steps

The following sections will walk you through the steps to configure the Adobe-AEM-Brightcove Connector. Note that before completing these steps, the connector must be installed - see Adobe-AEM-Brightcove Connector: Installation

Disable DAM transcoding

Unless you need DAM transcoding for non-Brightcove videos, you should turn it off to save processing time and storage:

  1. Open the AEM tools:
  2. Select Workflow:
  3. Select Models:
  4. Select List View:
  5. Select DAM Update Asset:
  6. Select Edit:
  7. Double-Click FFmpeg Transcoding from the workflow to edit it:
  8. Select the Process tab in the Step Properties dialog:
  9. Change the Process to No Operation:
  10. Change Ok to close the dialog and then save the changes

Enter account information

Now you are ready to configure the AEM-Brightcove connector with Video Cloud account information.

  1. Open the AEM Web Console configuration manager (http://localhost:4502/system/console/configMgr)
  2. Search on Brightcove Service to locate the service:
  3. Click on the + sign to add a new configuration:
  4. In the dialog, enter values as shown below. See Managing API Authentication Credentials for instructions on getting a client id and client secret. The permissions you need here are:
    • Account Alias: enter a name for the account to be displayed in the Connector
    • Account ID: enter your Brightcove publisher ID
    • Client ID: enter your Brightcove client ID from the Brightcove API Authentication page
    • Client Secret: enter your Brightcove client Secret
    • Allowed Groups: specify the group that will access the Connector
    • Players Store Path: keep default value
    • Default Video Player ID: keep default value or choose another player ID
    • Default Video Player Key: deprecated, leave blank or use value from previous connector
    • Default Playlist Player ID: enter a player ID enabled for playlists if you plan to use playlists
    • Default Playlist Player Key: deprecated, leave blank or use value from previous connector
    • Proxy server: leave blank or enter proxy if you were previously using a proxy
    • Dam Integration Path: keep default value
    • Default Asset Thumbnail PNG: keep default value
    • Test to make sure you access the Brightcove videos by going to http://localhost:4502/brightcove/admin - if you do not see a list of videos, go to the Security section (http://localhost:4502/useradmin#) and make sure that the user you are logged in as is a member of one of the Allowed Groups you set in the connector configuration.
  5. If you will be using multiple Video Cloud accounts, repeat the previous steps for each account.

Set up Brightcove Players

The last configuration task is to set up Brightcove Players.

  1. Navigate to the Site Admin tools (http://localhost:4502/siteadmin#/content in the Classic UI or http://localhost:4502/sites.html/content in the Touch UI)
  2. In the top-level Websites folder, create new folder called brightcovetools/li>
  3. Inside the new brightcovetools folder, create another new folder called players (we are creating a path to match the default Player Stores Path in the account settings configured in the previous section)
  4. Inside the players folder create a new Brightcove Player Page, selecting the type of embed you wish to use:
    If you are not sure which embed type you want to use, see Choosing the Correct Embed Code. The decision depends primarily on how much and what kind of player customizations you use.
  5. After you create the page, righ-click on it and select Properties from the menu (in the Classic UI; in the Touch UI, select the page and then click on Properties).
  6. Go the Brightcove Basic tab of the dialog, and select a player from the list:
  7. If you also want to use playlists, repeat the steps above to create a page for a Playlist Player.
  8. If you are using multiple Brightcove accounts, you will need to create player pages for each one.

You have now completed the configuration of the AEM-Brightcove Connector. Proceed to Using the Connector.
