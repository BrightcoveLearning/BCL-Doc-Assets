This app creates an RSS feed for iTunes from a collection of videos in your account. Because iTunes accepts either a JSON feed or a tradition XML RSS feed, we will use JSON here, since it is much easier to work with.

Genre for the feed

Apple requires that the items in the feed have a genre chosen from the available iTunes genres. In most cases, all the items in the feed will belong to the same genre.

Since the iTunes genre is not part of the standard Video Cloud video metadata, this app will assign a genre in one of two ways:

    If each of the videos have a custom field with the internal name genreid, the app will look that genre up to get the details and use those for the feed.
  • Otherwise, the app will present a list of movie and TV genres and allow you to select one that will be used for all of the videos in the feed.

Get credentials

To use the CMS API you will need proper credentials.

The easiest way to get credentials in most cases is through the Studio Admin API Authentication section (requires admin permissions on your account). See Managing API Authentication Credentials for details. In most cases, you probably just want to get permissions for all CMS API operation:

If the permissions you need are not available in Studio, or if you prefer to get them directly from the OAuth API, use your choice of the Get Client Credentials documents listed below. Whichever option you choose, you will need to ask for the correct operation permissions. The following can be used with cURL or Postman to get all permissions for the CMS API:

"operations": [
  "video-cloud/video/all"
]
  • OAuth: Get Client Credentials Using cURL
  • OAuth: Get Client Credentials Using Postman
